# 施夷光 QQ 智能机器人 — V2 版本说明书

---

## 一、V1 回顾

### 1.1 项目定位

基于 **Claude Code + cc-connect + NapCatQQ** 搭建的王者荣耀「西施」角色扮演 QQ 机器人。用户通过 QQ 与西施直接对话，由 Claude 大模型驱动角色人格进行回复。

### 1.2 技术架构

```
QQ 用户 → NapCat QQ 客户端 ──→ Python 代理 (:3002) ──→ cc-connect ──→ Claude Code
                                      ↑                      ↑              ↑
                                 过滤杂讯/排队提醒      多项目管理       AI 大脑
                                                       Telegram 编程助手  西施人格
```

| 组件                   | 作用              | 版本     |
| ---------------------- | ----------------- | -------- |
| Claude Code            | AI 引擎           | v2.1.126 |
| cc-connect             | IM ↔ AI 桥接     | v1.3.2   |
| NapCatQQ               | QQ 客户端协议适配 | v9.9.26  |
| Python WebSocket Proxy | 消息过滤代理      | —       |
| Windows 11             | 运行环境          | —       |

### 1.3 V1 已实现功能

- QQ 一对一私聊角色扮演
- 西施人设驱动对话（纯 Prompt，无工具调用）
- Python 代理过滤杂讯（排队提醒、思考过程、上下文标记、工作目录页脚）
- 两大项目并行运行（QQ 西施bot + Telegram 编程助手）
- 权限 bypass 模式 + 工具白名单限制
- `/new` 命令热刷新人设，无需重启
- 120 分钟空闲自动重置会话

### 1.5 V1 已知限制

- 无记忆系统：每次刷新会话丢失所有上下文
- 纯文本对话：不支持图片、语音、表情包
- 仅私聊：不支持群聊
- 无定时任务：不能主动发早安晚安
- 无情绪/好感度系统
- 人设更新需手动改文件 + `/new`

## 二、V2 目标与范围

### 2.1 产品定位

西施 bot 从"被动问答角色扮演"升级为 **"像真人在用 QQ 的聊天陪伴机器人"**：

- **记住每一个朋友** — 跨会话保留用户信息、喜好、聊天历史
- **有自己的情绪** — 好感度随互动变化，影响语气与亲密度
- **主动联系你** — 早安晚安、随机搭话、长时间不见会想你
- **发图斗嘴** — 对话中自然插入表情包，更像 QQ 聊天
- **开发者可见** — Web 面板阅览对话、分析数据、管理 bot

### 2.2 关键决策

| # | 决策项     | 选择                                                    |
| - | ---------- | ------------------------------------------------------- |
| 1 | 部署方案   | 云端 headless 渲染二维码（V2 本地开发，功能稳定后上云） |
| 2 | 用户规模   | 亲友小圈子（数十人），SQLite 足够                       |
| 3 | 主动对话   | 定时触发 + 随机搭话 + 长时间未回复提醒，每人每天 ≤2 条 |
| 4 | 记忆深度   | L1-L5 全保留（名字→事实→喜好→对话摘要→情感状态）    |
| 5 | 情绪可见性 | 部分可见（情绪可自然表达，好感度数值隐藏）              |
| 6 | 多媒体     | V2 先做表情包/图片                                      |
| 7 | 开发者面板 | Web 面板（浏览器）                                      |

### 2.3 记忆层次定义

| 层次 | 名称     | 内容               | 示例                                |
| ---- | -------- | ------------------ | ----------------------------------- |
| L1   | 身份     | 用户名字、称呼偏好 | "叫我小杰就行"                      |
| L2   | 事实     | 用户主动告知的信息 | "我生日 7 月 15"、"在北京工作"      |
| L3   | 喜好推断 | 从对话中归纳的特征 | 兴趣：游戏、摄影；性格：活泼        |
| L4   | 历史摘要 | 定期生成的对话总结 | "上周聊过他的期末考试，数学有点难"  |
| L5   | 情感关系 | 好感度 + 情绪状态  | 好感度 0.35（正向）、当前心情：活泼 |

## 三、V2 技术架构

### 3.1 架构总览

```
V1: QQ → NapCat(:3001) → proxy.py → (:3002) cc-connect → Claude Code

V2: QQ → NapCat(:3001) → V2 Engine(:3002) → cc-connect → Claude Code
              ↑ HTTP :4000        │
              │                   ├── SQLite DB（记忆/情绪/对话记录）
              └───────────────────├── Scheduler（主动消息）
                                  ├── Sticker Matcher（表情包匹配）
                                  └── FastAPI :8080（开发者面板）
```

**核心设计原则**：西施的 Claude Code 会话是只读的（无 Write/Edit/Bash 权限）—— 所有有状态逻辑（记忆、情绪、调度、表情包）在 V2 Engine 中运行。Claude Code 和 cc-connect 配置几乎不变。

### 3.2 V2 Engine 模块

```
v2-engine/
├── main.py              # 入口，启动全部服务
├── config.py            # 配置（端口、路径、API密钥）
├── router.py            # 消息路由核心（拦截进出双向消息）
├── filter.py            # 从 V1 proxy.py 移植的杂讯过滤
├── context/
│   └── builder.py       # 拼接记忆上下文注入到消息中
├── memory/
│   ├── manager.py       # L1-L5 记忆管理
│   └── summarizer.py    # 调用 Deepseek API 生成对话摘要(L4)
├── emotion/
│   └── engine.py        # 好感度评分 + 情绪状态机
├── stickers/
│   ├── library.py       # 表情包索引
│   ├── matcher.py       # 关键词+情绪匹配
│   └── files/           # 表情包 PNG 文件（20-30个）
├── scheduler/
│   └── scheduler.py     # APScheduler 定时/随机主动消息
├── napcat/
│   └── client.py        # NapCat HTTP API 客户端
├── db/
│   ├── database.py      # SQLite 初始化与迁移
│   └── models.py        # 表定义（9张表）
└── web/
    ├── app.py           # FastAPI 应用
    ├── routes.py        # API 路由
    └── templates/       # 前端页面模板
```

### 3.3 消息流

**入站（用户 → 西施）：**

1. NapCat WS → V2 Engine 收到消息
2. 解析用户 QQ 号 → 查 DB：身份(L1)、事实(L2)、喜好(L3)、摘要(L4)、情绪(L5)
3. 拼接触发上下文块（≤300 tokens）→ 注入到用户消息前方
4. 拦截开发者命令（`/记住` `/忘记` `/我是谁`）
5. 转发给 cc-connect → Claude 西施读取上下文后生成回复

**出站（西施 → 用户）：**

1. cc-connect → V2 Engine 收到西施回复
2. Filter 清洗杂讯（思考标记、上下文指示器等 V1 规则）
3. Sticker Matcher 分析回复文本 → 关键词+情绪匹配表情包
4. 若匹配成功 → 在回复前插入图片段
5. 存储消息到 DB → 转发给 NapCat

**主动消息（西施 → 用户）：**

1. Scheduler 按 cron/随机间隔触发
2. V2 Engine 调用 Deepseek API 生成 2-3 句消息（使用紧凑 prompt）
3. 通过 NapCat HTTP API 直接发送（绕过 cc-connect）
4. 记录到 DB

### 3.4 上下文注入格式

```
[系统记忆]
用户: {name} | 称呼: {preferred_name}
心情: {mood} | 好感度: {正向/中性/需要关心}
[Ta告诉过你]: {fact1}; {fact2}; {fact3}
[你感觉Ta]: {insight1}; {insight2}
[最近聊过]: {summary1} {summary2}
---
{用户原始消息}
```

### 3.5 开发者命令

| 命令               | 功能               | 处理方式                               |
| ------------------ | ------------------ | -------------------------------------- |
| `/记住 <事实>`   | 存储一条事实       | V2 Engine 拦截 → 写入 DB → 返回确认  |
| `/忘记 <关键词>` | 删除匹配的事实     | V2 Engine 拦截 → 删除匹配 → 返回确认 |
| `/我是谁`        | 查看西施对你的记忆 | V2 Engine 拦截 → 查询返回             |
| `/new`           | 刷新 Claude 会话   | 转发给 cc-connect（V1 行为不变）       |

### 3.6 情绪系统

| 参数     | 范围                                                         | 可见性           |
| -------- | ------------------------------------------------------------ | ---------------- |
| 好感度   | -1.0 ~ 1.0，起始 0.1                                         | **隐藏**   |
| 当前心情 | happy / playful / calm / shy / annoyed / thoughtful / sleepy | **可感知** |

- 好感度变化：回复长度比、回复延迟、正面/负面关键词、互动频率
- 心情影响：语气亲密度、表情包选择、回复风格
- 衰减：7 天不活跃微量衰减

## 七、工作流（ECC 标准）

### 开发流程

每个 Phase 按以下流程执行：

```
1. 研究（Research）
   - 搜索 GitHub 相似实现
   - 查阅 NapCat/OneBot 文档
   ↓
2. 规划（Plan）
   - 写具体任务清单
   - 确认接口与数据流
   ↓
3. TDD（测试驱动）
   - 先写测试（RED）
   - 实现功能（GREEN）
   - 重构优化（IMPROVE）
   ↓
4. 审查（Review）
   - 代码质量检查
   - 安全检查
   ↓
5. 提交（Commit）
   - 详细 commit message
```

### 文件组织原则

- 每个模块 ≤400 行，超过则拆分
- 函数 ≤50 行
- 嵌套 ≤4 层
- 不可变数据模式
- 输入在系统边界校验

### 启动顺序（Phase 0+）

```
1. VPN（如需要）
2. NapCat QQ（扫码登录）
3. python v2-engine/main.py（替代原 proxy.py）
4. cc-connect（连接 :3002）
5. 访问 http://localhost:8080（开发者面板）
```
